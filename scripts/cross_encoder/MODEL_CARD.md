# Hindsight Memory Reranker

A fine-tuned cross-encoder reranking model optimized for ranking documents in Hindsight-formatted agent memory systems.

## Model Description

This model is a fine-tuned version of [`cross-encoder/ms-marco-MiniLM-L12-v2`](https://huggingface.co/cross-encoder/ms-marco-MiniLM-L12-v2), specifically trained to rerank memory documents formatted according to the [Hindsight](https://arxiv.org/abs/2512.12818) memory architecture.

The model is trained with Quantization-Aware Training (QAT) and exported to ONNX format with INT8 dynamic activation and INT4 weight quantization for efficient inference.

### Key Features

- **Hindsight-Optimized**: Trained on documents with temporal anchors, fact types, and contextual metadata
- **Quantized**: INT8/INT4 quantization for efficient deployment
- **ONNX Export**: Ready for production deployment

## Usage

### With ONNX Runtime

```python
import onnxruntime as ort
from transformers import AutoTokenizer

tokenizer = AutoTokenizer.from_pretrained("your-username/hindsight-memory-reranker")
session = ort.InferenceSession("model.onnx")

def rerank(query: str, documents: list[str]) -> list[float]:
    scores = []
    for doc in documents:
        inputs = tokenizer(query, doc, padding=True, truncation=True, max_length=512, return_tensors="np")
        score = session.run(None, inputs)[0]
        scores.append(float(score[0]))
    return scores
```

### Document Formatting

Documents should be formatted in Hindsight format before ranking:

```
[Date: January 14, 2026 (2026-01-14)] [Experience] Architecture Decision: We deprecated the Redis layer because the maintenance overhead outweighed the benefits.
```

Format template:
```
[Date: {Month DD, YYYY (YYYY-MM-DD)}] [{Type}] {Context}: {Narrative Text}
```

Where:
- **Type**: `Experience`, `Opinion`, `World`, or `Observation`
- **Context**: Category label (e.g., "Architecture Decision", "Project Planning")
- **Date**: ISO timestamp for temporal awareness

## Training Details

### Training Data

Synthetic data generated by Gemini for fine-tuning the reranker on Hindsight-formatted memory documents.

| Split | Samples |
|-------|---------|
| Train | 300 |
| Eval | 70 |
| Test | 60 |

Each training example is a triplet containing:
- `query`: Natural language question
- `positive`: Relevant Hindsight-formatted document
- `negative`: Distractor document (similar but irrelevant)

Example:
```json
{
  "query": "Why did we drop the Redis cache?",
  "positive": {
    "text": "We deprecated the Redis layer because the maintenance overhead of the cluster outweighed the latency benefits for our specific read patterns.",
    "occurred_start": "2025-08-14T10:00:00",
    "type": "experience",
    "context": "Architecture Decision"
  },
  "negative": {
    "text": "I set up a Redis instance on Friday to test the new session management library.",
    "occurred_start": "2025-08-10T09:00:00",
    "type": "experience",
    "context": "Experiment"
  }
}
```

### Training Configuration

| Parameter | Value |
|-----------|-------|
| Base Model | cross-encoder/ms-marco-MiniLM-L12-v2 |
| Epochs | 10 |
| Batch Size | 8 |
| Learning Rate | 6e-6 |
| Loss Function | Margin Ranking Loss (margin=1.0) |
| Max Sequence Length | 512 |
| Quantization | INT8 dynamic activation, INT4 weights (QAT) |

### Loss Function

Uses Margin Ranking Loss to ensure positive documents receive higher relevance scores than negative documents:

```
loss = max(0, margin - score_positive + score_negative)
```

## Model Architecture

- **Type**: Cross-Encoder for sequence classification
- **Base**: MiniLM-L12 (12 transformer layers)
- **Output**: Single relevance score (logits)
- **Quantization**: INT8 dynamic activation + INT4 weight (via torchao)

## Limitations

- Trained on synthetic data; may require domain-specific fine-tuning for production use
- Optimized for Hindsight document format; may not generalize to arbitrary document formats
- English language only
